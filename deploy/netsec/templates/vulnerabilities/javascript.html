漏洞代码分析
// 漏洞1：Token由用户名和固定盐值生成
var fixedSalt = "SaltForToken_v1";  // 硬编码盐值

function generateToken() {
    var username = document.getElementById("username").value;
    var token = md5(username + fixedSalt);  // 可预测
    return username + ":" + token;
}

// 漏洞2：客户端验证可被绕过
function verifyToken(token) {
    var parts = token.split(":");
    if (parts.length === 2) return true;  // 弱验证
    return false;
}
测试步骤
  1. 分析JavaScript代码逻辑
  2. 找出Token生成算法（用户名 + 固定盐值）
  3. 使用相同算法为其他用户生成Token
  4. 观察：盐值硬编码在代码中，任何人都能看到
漏洞说明
漏洞原理

前端JavaScript代码中可能存在多种安全漏洞，如敏感信息暴露、客户端验证、弱加密等。

常见问题
  • 敏感信息硬编码
  • 客户端验证可绕过
  • 弱加密算法
  • Token在客户端生成
解题提示

查看页面源码，分析JS代码逻辑

防护措施
  • 不在前端存储敏感信息
  • 服务端生成Token
  • 服务端验证
  • 使用强加密算法
  • 混淆压缩JavaScript代码
