测试步骤
  1. 创建一个测试文件（如 test.txt 或 shell.php）
  2. 通过上传表单上传该文件
  3. 观察：文件被直接保存，没有类型检查
  4. 在真实场景中，可上传 Webshell 获取服务器控制权
漏洞说明
漏洞原理

文件上传漏洞允许攻击者上传恶意文件到服务器，并可能通过访问这些文件来执行任意代码。

危害
  • 服务器被完全控制
  • Webshell后门
  • 敏感数据泄露
解题提示

上传任意文件，没有文件类型限制

防护措施
  • 白名单验证文件类型
  • 检查文件内容（魔数）
  • 重命名上传文件
  • 禁止执行上传目录的文件
  • 限制文件大小
