快速上手：SQL 注入获取所有数据
第1步：正常查询

先输入正常 ID 看效果

输入: 1
会返回用户 ID=1 的信息
第2步：注入 Payload

替换为 SQL 注入语句

1 OR 1=1 👉 这会返回所有用户！
第3步：查看结果

点击"查询"按钮

返回多行数据表格
看到多余数据 = 注入成功！
Payload 参考
  • 1 OR 1=1 — 返回所有用户
  • 1 UNION SELECT 1,username,password_hash,1 FROM users — 联合查询
  • 1; DROP TABLE users-- — 堆叠查询（危险）
漏洞说明
漏洞原理

用户输入直接拼接到SQL语句中执行，没有参数化查询。

利用条件
  • 用户输入直接拼接到SQL
  • 没有输入过滤或验证
  • 没有使用参数化查询
解题提示

尝试使用单引号或OR 1=1绕过认证或获取所有数据

防护措施
  • 使用参数化查询
  • 输入验证和过滤
  • 限制数据库权限
  • 使用ORM框架
  • 错误信息不显示给用户
